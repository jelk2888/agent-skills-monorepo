## 2. Phase 1: 问题解析

### 2.1 问题分类

| 问题类型 | 特征 | 调研重点 |
|---------|------|---------|
| 概念理解型 | "什么是XX？" | 定义、直觉、历史、应用 |
| 定理证明型 | "为什么XX成立？" | 证明思路、关键引理、反例 |
| 计算方法型 | "怎么求XX？" | 算法步骤、数值方法、实例 |
| 应用场景型 | "XX有什么用？" | 实际应用、工程案例、前沿进展 |
| 对比辨析型 | "XX和YY有什么区别？" | 本质差异、联系、统一视角 |
| 历史发展型 | "XX是怎么被发现的？" | 发现历程、关键人物、范式转变 |

### 2.2 关键词提取策略

```
用户问题: "傅里叶变换的本质是什么？"

提取关键词:
├── 核心: 傅里叶变换, 本质
├── 扩展: 频域, 时频分析, 正交基, 谐波分解
├── 关联: 傅里叶级数, 拉普拉斯变换, 离散傅里叶变换
└── 应用: 信号处理, 图像压缩, 量子力学

搜索策略:
├── 学术: "Fourier transform intuition" / "傅里叶变换 几何直觉"
├── 教学: "3b1b Fourier" / "傅里叶变换 可视化"
├── 历史: "Fourier heat equation history" / "傅里叶 热传导"
└── 应用: "FFT applications" / "傅里叶变换 应用"
```

## 3. Phase 2: 网络调研

### 3.1 调研维度

对每个问题，从以下5个维度进行调研：

| 维度 | 搜索目标 | 信息类型 |
|------|---------|---------|
| **定理/定律** | 核心数学表述、证明、适用条件 | 公式、定理陈述、证明概要 |
| **历史/发现** | 概念的起源、发展历程、关键人物 | 故事、时间线、轶事 |
| **实验/现象** | 支撑理论的经典实验、可观测现象 | 实验描述、数据、可视化 |
| **经典例题** | 教科书级例题、竞赛题、应用题 | 题目+解法、分步推导 |
| **教学资源** | 优质教学文章、视频、可视化 | 教学方法、常见误解、最佳类比 |

### 3.2 搜索策略模板

```
搜索第1轮: 广度搜索（了解全貌）
├── "{概念} 入门/直觉/intuition"
├── "{概念} 可视化/visualization"
└── "what is {concept} really"

搜索第2轮: 深度搜索（核心定理/证明）
├── "{概念} 定理/证明/proof"
├── "{概念} 推导/derivation"
└── "{概念} 为什么成立/why it works"

搜索第3轮: 应用搜索（经典例题/实验）
├── "{概念} 经典例题/examples"
├── "{概念} 实验/experiment"
└── "{概念} 应用/application"

搜索第4轮: 教学搜索（最佳讲解方式）
├── "{概念} 教学方法/teaching"
├── "{概念} 常见误解/misconception"
└── "best way to explain {concept}"
```

### 3.3 信息来源优先级

| 优先级 | 来源类型 | 可信度 | 示例 |
|--------|---------|--------|------|
| ★★★★★ | 学术论文/教科书 | 最高 | arXiv, MIT OCW, 教科书 |
| ★★★★☆ | 权威教育平台 | 高 | Khan Academy, 3Blue1Brown, Brilliant |
| ★★★★☆ | 大学课程资料 | 高 | MIT OCW, Stanford CS, 清华学堂 |
| ★★★☆☆ | 技术博客/社区 | 中高 | Math StackExchange, Zhihu专栏 |
| ★★★☆☆ | 百科全书 | 中 | Wikipedia, MathWorld, Scholarpedia |
| ★★☆☆☆ | 个人博客/视频 | 中低 | B站, YouTube, 个人网站 |
| ★☆☆☆☆ | AI生成内容 | 低 | 需交叉验证 |

## 4. Phase 3: 信息提炼

### 4.1 筛选标准

对搜索到的每条信息，按以下标准评估：

```python
class InfoQuality:
    relevance: int       # 与核心问题的相关度 (1-5)
    authority: int       # 来源权威性 (1-5)
    accessibility: int   # 对目标观众的可理解性 (1-5)
    visualizable: int    # 可转化为动画的程度 (1-5)
    novelty: int         # 信息新颖度/独特视角 (1-5)
    
    @property
    def score(self):
        return (
            self.relevance * 3 +
            self.authority * 2 +
            self.accessibility * 2 +
            self.visualizable * 2 +
            self.novelty * 1
        ) / 10
```

**保留标准**: score ≥ 3.0 的信息纳入脉络构建

### 4.2 去重与交叉验证

```
信息A: "傅里叶变换是将信号分解为频率成分"
信息B: "傅里叶变换是时域到频域的映射"
信息C: "傅里叶变换是用正弦波逼近任意信号"

→ 合并为: "傅里叶变换的本质：将任意信号分解为不同频率的正弦波叠加（时域→频域）"

交叉验证:
├── 3个来源一致 → ✅ 高置信度
├── 2个来源一致 + 1个不同 → ⚠️ 标注分歧
└── 来源互相矛盾 → ❌ 需要更权威来源裁定
```

### 4.3 常见误解收集

每个概念至少收集1-2个常见误解，这是教学视频的关键素材：

| 概念 | 常见误解 | 正确理解 |
|------|---------|---------|
| 导数 | "导数就是斜率" | 导数是变化率的极限，斜率只是几何表现 |
| 积分 | "积分就是面积" | 面积只是几何直观，本质是累积量 |
| 特征值 | "特征值就是矩阵的大小" | 特征值是变换在特定方向上的缩放比例 |
| 傅里叶变换 | "傅里叶变换只处理周期信号" | 傅里叶变换可处理任意可积信号 |
| 概率 | "独立=互不影响" | 独立是P(AB)=P(A)P(B)，不是"没有关系" |

## 5. Phase 4: 脉络构建

### 5.1 教学叙事线模板

```
┌────────────────────────────────────────────────────────┐
│  开场: 悬念/反直觉现象 (来自调研中的历史/实验)          │
│  例: "1807年，傅里叶声称任意函数都能用三角级数表示，    │
│       当时整个法国数学界都认为他疯了。"                  │
├────────────────────────────────────────────────────────┤
│  第一幕: 直觉建立 (来自调研中的可视化/类比)             │
│  ├─ 用最简单的例子展示核心思想                          │
│  ├─ 先看到现象，再引入术语                              │
│  └─ 例: 用旋转圆上的点画图 → "这就是傅里叶变换在做的事" │
├────────────────────────────────────────────────────────┤
│  第二幕: 形式化 (来自调研中的定理/证明)                 │
│  ├─ 引入精确的数学表述                                  │
│  ├─ 展示关键定理和证明思路                              │
│  └─ 例: 正交基分解 → 内积提取系数 → 傅里叶系数公式      │
├────────────────────────────────────────────────────────┤
│  第三幕: 深入理解 (来自调研中的经典例题/应用)           │
│  ├─ 经典例题演示                                       │
│  ├─ 常见误解澄清                                       │
│  └─ 例: 方波→正弦波叠加 / 信号滤波 / JPEG压缩          │
├────────────────────────────────────────────────────────┤
│  收尾: 回扣开场 (来自调研中的历史/前沿)                 │
│  ├─ "现在我们知道傅里叶并没有疯"                       │
│  └─ 指向更深的主题                                     │
└────────────────────────────────────────────────────────┘
```

### 5.2 ResearchReport 输出结构

```python
class ResearchReport:
    topic: str                    # 主题
    core_question: str            # 核心问题
    key_findings: list            # 关键发现列表
    
    theorems: list                # 相关定理/公式
    experiments: list             # 经典实验/现象
    classic_examples: list        # 经典例题
    historical_context: str       # 历史背景
    misconceptions: list          # 常见误解
    best_analogies: list          # 最佳类比/隐喻
    
    narrative_structure: dict     # 教学叙事结构
    visual_elements: list         # 可视化元素清单
    animation_suggestions: list   # 动画建议
    
    sources: list                 # 信息来源列表
    confidence_scores: dict       # 各信息的置信度
```

### 5.3 从调研到脚本的转化规则

| 调研元素 | 脚本转化 | 动画转化 |
|---------|---------|---------|
| 历史故事/轶事 | 开场悬念 | 时间线动画/人物出场 |
| 直觉/类比 | 第一幕直觉建立 | 隐喻可视化（查隐喻库2.6） |
| 定理/公式 | 第二幕形式化 | 逐步书写+几何对应 |
| 经典例题 | 第三幕深入 | 完整解题动画 |
| 常见误解 | 插入澄清 | 对比动画（错误vs正确） |
| 实验/现象 | 直觉佐证 | 物理模拟动画 |
| 应用场景 | 收尾拓展 | 应用展示动画 |

## 8. 信息质量评估标准

### 8.1 事实性评估

| 检查项 | 方法 |
|--------|------|
| 数学正确性 | 交叉验证2+权威来源 |
| 历史准确性 | 对照原始论文/传记 |
| 实验可重复性 | 查找原始实验数据 |
| 适用范围 | 检查定理的前提条件 |

### 8.2 教学性评估

| 检查项 | 标准 |
|--------|------|
| 可视化潜力 | 能否用Manim动画清晰展示？ |
| 认知负荷 | 目标观众能否在3分钟内理解核心？ |
| 误解风险 | 这个解释是否可能产生新的误解？ |
| 叙事价值 | 这个信息是否让故事更完整？ |

### 8.3 降级策略

当调研信息不足以支撑完整动画时：

| 情况 | 策略 |
|------|------|
| 找不到好的可视化角度 | 使用"分步推导动画"（策略B） |
| 概念过于抽象 | 使用"概念地图"（策略C） |
| 信息来源冲突 | 标注分歧，展示两种视角 |
| 缺乏经典例题 | 自行构造简单例子 |
| 历史资料不足 | 跳过历史，直接从直觉开始 |

## 9. 完整示例

### 示例：用户问"欧拉公式为什么被称为最美公式？"

#### Phase 1: 问题解析
- 核心概念: 欧拉公式 e^(iπ) + 1 = 0
- 学科: 复分析/微积分
- 关键词: 欧拉公式, 复指数, 单位圆, 最美公式

#### Phase 2: 网络调研
搜索结果摘要:
1. **定理**: e^(iθ) = cos(θ) + i·sin(θ) (欧拉公式一般形式)
2. **历史**: 欧拉1748年首次发表，但Roger Cotes更早发现等价形式
3. **直觉**: 复指数 = 单位圆上的旋转
4. **经典证明**: 泰勒展开对比法
5. **应用**: 信号处理、量子力学、电路分析

#### Phase 3: 信息提炼
- 核心洞察: e^(iθ) 描述的是复平面上角度θ的旋转
- 最佳类比: 乘以e^(iθ) = 在复平面上旋转θ角
- 常见误解: ❌ "i是虚数所以没有几何意义" → ✅ "i代表90°旋转"
- 可视化: 单位圆上点的运动 + 指数函数的旋转效果

#### Phase 4: 脉络构建

```
开场: "5个最重要的常数，用一个等号连在一起——这是数学界公认的'最美公式'"
第一幕: 先看1的旋转（0°），i的旋转（90°），-1的旋转（180°）
第二幕: e^(iθ) = cos(θ) + i·sin(θ) 的泰勒展开证明
第三幕: 令θ=π → e^(iπ) = -1 → e^(iπ) + 1 = 0
收尾: "5个常数，0(加法单位元) 1(乘法单位元) e(分析) π(几何) i(代数)，
       一个公式统一了数学的三大分支"
```

#### Phase 5: 生成的 Manim 代码框架

```python
class EulersFormula(Scene):
    def construct(self):
        self.show_opening()       # 开场悬念
        self.rotation_intuition() # 旋转直觉
        self.taylor_proof()       # 泰勒展开证明
        self.beautiful_moment()   # θ=π 的美妙瞬间
        self.conclusion()         # 统一三大分支
```

## 10. 研究员模式与标准模式的区别

| 对比项 | 标准模式 | 研究员模式 |
|--------|---------|-----------|
| 输入 | 用户的一句话描述 | 用户的问题/主题 |
| 第一步 | 提示词扩展 | 网络调研 |
| 信息来源 | 技能内置知识 | 网络搜索+交叉验证 |
