## 1. 心智模型详解

### 1.1 三层认知模型（如何让观众理解）

| 层级 | 名称 | 操作方式 | 例子 |
|------|------|---------|------|
| L1 | **直觉层** | 具体图像、物理隐喻 | 向量=箭头，矩阵=空间变形 |
| L2 | **连接层** | Transform变形、颜色关联 | 蓝色向量→绿色输出 |
| L3 | **形式层** | LaTeX公式、符号语言 | A**v** = λ**v** |

**关键**: 必须按 L1→L2→L3 顺序建立，不能跳步！

### 1.2 "Essence of X" 思维框架

这是3b1b所有视频系列的命名和设计模式：

```
Step 1: 找到 X 的本质问题 — "为什么X是这样定义的？"
Step 2: 用几何直觉回答 — 能不能用图像/运动解释？
Step 3: 建立概念网络 — X与哪些其他概念通过Transform相连？
Step 4: 形式化收尾 — 符号语言精确描述
Step 5: 回顾闭环 — 回到最初的问题，现在能回答了吗？
```

### 1.3 动画目的论判断树（详细版）

```
这个动画运动是否服务于理解？
├── 是 → 它是否是最简洁的表达方式？
│   ├── 是 → ✅ 保留
│   └── 否 → 用更简单的方式替代
│       ├── 能用静态图说明？ → 用静态图
│       ├── 能用更少帧数？ → 缩短 run_time
│       └── 能用更简单的动画类型？ → 替换（如 FadeIn 代替复杂入场）
└── 否 → ❌ 删除（无论多酷）
    ├── 仅仅为了"好看" → 删除
    ├── 重复已有信息 → 删除
    └── 分散注意力 → 删除
```

## 2. 色彩系统规范

### 2.1 语义色彩映射表

| 概念类别 | 推荐色 | Manim常量 | 使用场景 |
|---------|--------|-----------|---------|
| 向量/输入量 | 蓝色 | BLUE_C | 箭头、输入向量、初始状态 |
| 变换结果/输出 | 绿色/青绿 | TEAL_C | 变换后的向量、输出值 |
| 强调/警告 | 红/橙 | RED_C, ORANGE | 重要标注、错误提示 |
| 矩阵/变换操作 | 紫/品红 | PURPLE_C | 线性变换、算子、矩阵 |
| 特殊量 | 金黄色 | GOLD_C | 特征值、重要常数、重要发现 |
| 文本 | 白 | WHITE | 所有文本（固定） |
| 坐标轴/网格 | 灰 | GRAY_C | 辅助结构（不抢注意力）|

### 2.2 临时色（用于非重复出现的元素）

```python
class Colors3B1B:
    # ... (语义色见SKILL.md) ...

    # === 临时色 ===
    TEMP_A = GREEN_C
    TEMP_B = ORANGE
    TEMP_C = MAROON_C
```

### 2.3 配色铁律（详细说明）

1. **全局一致性**: 同一概念整个系列永远同色
   - 违反示例: 第1集向量用蓝色，第3集向量用绿色 ❌
   - 正确做法: 全系列向量始终用 BLUE_C ✅

2. **对比度优先**: 深色背景 + 高饱和度前景色
   - 背景色: `#1e1e2e`（接近黑色但带蓝调）
   - 避免纯黑 `#000000`（太硬）和纯白背景

3. **色彩克制**: 单帧画面 ≤ 4-5种颜色
   - 如果需要更多颜色 → 分批展示，不要同时出现

4. **暖冷对比**: 表示变化方向
   - 冷色(蓝/青) = 输入/变化前
   - 暖色(红/橙/黄) = 输出/变化后

5. **`_C` 后缀**: 始终使用 `BLUE_C` 而非 `BLUE`
   - `_C` 版本在深色背景上对比度更好

### 2.4 常见配色方案模板

**线性代数系列**:
```python
INPUT = BLUE_C       # 向量
OUTPUT = TEAL_C      # 变换结果
TRANSFORM = PURPLE_C # 矩阵/变换
HIGHLIGHT = GOLD_C   # 特征值
```

**微积分系列**:
```python
ORIGINAL = BLUE_C    # 原函数
DERIVATIVE = RED_C   # 导数
INTEGRAL = GREEN_C   # 积分
APPROX = YELLOW      # 近似
```

**概率统计系列**:
```python
DATA = BLUE_C        # 数据点
DISTRIBUTION = TEAL_C # 分布曲线
PARAMETER = GOLD_C   # 参数
OUTLIER = RED_C      # 异常值
```

## 3. 字体排印规范

### 3.1 完整字体选择指南

| 内容类型 | Mobject类 | 字体 | 示例 | 注意事项 |
|---------|-----------|------|------|---------|
| 章节标题 | `Tex()` | 加粗LaTeX | `Tex(r"\textbf{线性变换}")` | 大号，白色 |
| 正文说明 | `Text()` | Sans-serif | `Text("这意味着...")` | 正常字号 |
| 数学公式 | `MathTex()` | Latin Modern Math | `MathTex(r"\int_a^b f(x)dx")` | 不需要$包裹 |
| 行内公式 | `Tex()` | LaTeX混排 | `Tex(r"令 $f(x) = x^2$")` | 需要$包裹 |
| 图上标注 | `MathTex()` | 小号 | `MathTex(r"\vec{v}").scale(0.6)` | 0.5-0.7倍缩放 |
| 代码片段 | `Code()` | 等宽 | `Code("v = np.array([3,2])")` | 社区版专用 |
| 可变数字 | `DecimalNumber()` | — | `DecimalNumber(0, num_decimal_places=2)` | 配合ValueTracker |

### 3.2 中文文本处理

```python
# ✅ 正确: 中文用 Text()
title = Text("微积分基本定理", font_size=36)

# ✅ 正确: 指定中文字体
title = Text("中文内容", font="Noto Sans CJK SC")
title = Text("中文内容", font="Microsoft YaHei")

# ❌ 错误: 中文不要用 MathTex（可能乱码）
# title = MathTex("微积分基本定理")

# ✅ 正确: LaTeX中文需配置ctex
formula = Tex(r"\text{根据牛顿-莱布尼茨公式}", tex_template=TexTemplateLibrary.ctex)
```

### 3.3 公式着色技巧

```python
# 按子公式着色
formula = MathTex(r"\int_0^\infty", r"e^{-x^2}", r"dx", r"=", r"\frac{\sqrt{\pi}}{2}")
formula[0].set_color(RED)
formula[1].set_color(BLUE_C)

# 按关键词着色
text = Text("梯度下降法")
text.set_color_by_t2c({'梯度': BLUE_C, '下降': RED_C})

# 渐变效果
text = Text("彩虹文字")
text.set_color_by_gradient(RED, ORANGE, YELLOW, GREEN, BLUE_C, PURPLE_C)
```

## 4. 动画节奏规范

### 4.1 详细时长参数

| 动作类型 | run_time | 说明 | 何时调整 |
|---------|----------|------|---------|
| FadeIn / FadeOut | 0.5-0.8s | 干脆利落 | 简单对象用0.5，复杂对象用0.8 |
| Create | 1.0-2.0s | 取决于复杂度 | 简单图形1.0，复杂图形2.0 |
| Write | 0.08s/字符 | 模拟手写速度 | 长文本可加快到0.05s/字符 |
| GrowFromCenter | 0.6-1.0s | 自然生长感 | — |
| Transform | 1.0-1.5s | 平滑过渡 | 重要变形延长到2.0s |
| ReplacementTransform | 1.0-1.5s | 概念演变 | 核心洞见时刻延长到2.0s |
| MoveTo / Shift | 0.8-1.2s | 不太快不太慢 | — |
| Rotate | 1.0-1.5s | 自然转动 | — |
| **wait 停顿** | **0.5-1.5s** | **★最重要！** | **关键洞察后用1.5s** |

### 4.2 呼吸节奏模板

```
标准节奏:
[信息输入 2-3s] → [停顿 0.5-1s] → [变形连接 1-2s] → [停顿 0.5-1s]

关键洞见节奏:
[铺垫 2-3s] → [变形 1.5s] → [长停顿 1.5s] → [强调动画 0.5s] → [停顿 1.0s]

开场节奏:
[问题提出 2s] → [停顿 1.0s] → [第一个视觉 2s] → [停顿 0.5s]
```

### 4.3 信息密度规则

```python
class Timing:
    MAX_ACTIVE_ELEMENTS = 5      # 一帧最多活跃元素
    NEW_INFO_INTERVAL = 4.0      # 新信息最小间隔（秒）
    SIMULTANEOUS_MOTION = 3      # 同时运动的物体上限
```

**违反密度规则的信号**:
- 观众需要暂停才能看清 → 信息太密
- 画面静止超过3秒无事发生 → 信息太稀
- 同时有4+个对象在运动 → 认知超载

## 5. 过渡动画选择指南

### 5.1 详细过渡决策树

```
需要切换内容？
├── 新内容与旧内容相关？
│   ├── 是 → 同一对象的变形？
│   │   ├── 是 → Transform(A, B)          # A变为B的外观
│   │   └── 否 → ReplacementTransform(A, B) # A消失，B出现
│   └── 否 → 公式/文字重组？
│       ├── 是 → TransformMatchingTex/Sshapes
│       └── 否 → FadeOut A + FadeIn B
├── 需要展示空间关系？
│   └── camera.frame.animate.move_to()
└── 需要强调某元素？
    ├── 短暂注意 → Indicate()
    ├── 持续关注 → SurroundingRectangle()
    └── 空间聚焦 → FocusOn()
```

### 5.2 跨Scene切换

- 硬切或淡出到黑色再淡入
- 极少花哨转场
- 保持色彩一致性是跨Scene连贯性的关键

## 6. 隐喻库（完整版）

### 6.1 数学概念隐喻

| 数学概念 | 推荐隐喻 | 视觉实现 | 经典出处 | 难度 |
|---------|---------|---------|---------|------|
| 线性变换 | 空间拉伸/旋转/剪切 | 网格变形动画 | 线性代数 ★★★ | 中 |
| 导数 | 显微镜下斜率连续变化 | ZoomedScene连续缩放 | 微积分 ★★★ | 中 |
| 积分 | 累积切片求和 | 面积切片堆叠动画 | 微积分 ★★★ | 中 |
| 傅里叶变换 | 多旋转矢量合成波形 | Epicycle嵌套圆 | 傅里叶 ★★★ | 高 |
| 特征值 | 不被改变方向的"特殊轴" | 特殊方向高亮脉冲 | 线性代数 ★★☆ | 中 |
| 行列式 | 面积/体积缩放比例 | 平行四边形面积实时变化 | 线性代数 ★★★ | 中 |
| 矩阵乘法 | 复合变换（两次连续变形） | 连续网格变形叠加 | 线性代数 ★★★ | 中 |
| 梯度下降 | 球在地形图上滚下山坡 | 3D损失地形+球体滚动 | 神经网络 ★★☆ | 高 |
| 泰勒展开 | 多项式逐项逼近曲线 | 逐项叠加拟合过程 | 微积分 ★★☆ | 中 |
| 卷积 | 翻转+滑动+重叠求和 | 函数翻转滑动积分 | 信号处理 ★☆☆ | 高 |
| 基变换 | 坐标系旋转变换 | 两套坐标网的叠加变换 | 线性代数 ★★☆ | 中 |
| 点积 | 投影长度 | 向量投影到另一向量的动画 | 线性代数 ★★☆ | 低 |
| 链式法则 | 复合函数的传递效应 | 齿轮嵌套或流水线 | 微积分 ★☆☆ | 中 |

### 6.2 物理概念隐喻

| 物理概念 | 推荐隐喻 | 视觉实现 |
|---------|---------|---------|
| 抛体运动 | 球的飞行轨迹 | ValueTracker + 轨迹线 |
| 波动 | 水面涟漪传播 | 参数曲线 + 传播动画 |
| 电磁场 | 场线/力线 | 箭头场 + 流线 |
| 热传导 | 颜色渐变扩散 | 颜色插值 + 区域扩展 |
| 光学折射 | 光线路径弯折 | 光线 + 法线 + 角度标注 |

### 6.3 使用新隐喻的原则

1. **数学正确性**: 必须数学上正确（不能只好看而误导）
2. **简单性**: 越简单的物理类比越好（日常经验 > 抽象模型）
3. **单一性**: 一个视频中隐喻类型 ≤ 2种（避免混乱）
4. **一致性**: 同一系列中同一概念用同一隐喻

### 6.4 当没有现成隐喻时

```
Step 1: 问"这个概念在物理世界中最接近什么？"
Step 2: 如果找不到物理类比 → 尝试几何类比
Step 3: 如果几何类比也找不到 → 使用降级策略（见SKILL.md第十一节）
Step 4: 永远不要为了"好看"而使用不准确的隐喻
```

## 7. 质量检核清单（详细版）

### A. 技术层面

- [ ] 所有Scene可正常渲染无报错
- [ ] 分辨率和帧率符合目标规格
- [ ] 色彩在全视频中保持一致（对照config.py）
- [ ] 无 unintended artifacts（闪烁/撕裂/错位/重叠）
- [ ] 文字清晰可读（不过小不过密）
- [ ] LaTeX公式正确渲染（无编译错误）
- [ ] Updater在场景结束时已清除

### B. 设计层面（3b1b风味检测）

- [ ] 单帧信息密度 ≤ 5个活跃元素
- [ ] 每个动画运动都有明确的教学目的（通过目的论判断树）
- [ ] 节奏自然（有呼吸感，有停顿，不是一直动不停）
- [ ] 颜色编码全局一致且符合语义
- [ ] 过渡方式恰当（相关→Transform，无关→Fade）
- [ ] **深色背景**（不是白色！）
- [ ] 使用 `_C` 后缀颜色常量
- [ ] 每引入公式前有对应的几何图像

### C. 教学层面

- [ ] 前5秒内抓住注意力（提问/反直觉现象）
- [ ] 遵循 直觉→形式化 的顺序（不反着来）
- [ ] 关键结论有足够的铺垫和强调
- [ ] 结尾回扣开头的问题
- [ ] 目标观众能跟上（不过快也不拖沓）
- [ ] 没有裸公式单独出现超过3秒
- [ ] ReplacementTransform用于展示概念联系

### D. 音频层面

- [ ] 配音语速与动画节奏同步
- [ ] 无过长静音或音频空白
- [ ] BGM不干扰人声（-20dB左右）
- [ ] 关键术语发音正确
